Preguntas

  1. ¿Qué es el DOM?

    2. El DOM (Document Object Model) es una representación en memoria de la estructura de un documento HTML o XML. Proporciona una interfaz para interactuar y manipular los elementos de un documento web.

  2. ¿Qué es un nodo?

    3. Un nodo en el contexto del DOM es cualquier objeto dentro de la estructura del documento. Cada elemento HTML, atributo, texto, comentario y otros tipos de contenido se representan como nodos en el DOM. Los nodos están interconectados formando un árbol jerárquico.

  3. ¿Cómo se accede a un elemento del DOM?

    4. Para acceder a un elemento del DOM, se utilizan métodos de acceso proporcionados por el lenguaje JavaScript. Algunos métodos comunes son:

    getElementById: Permite acceder a un elemento utilizando su ID único.
    getElementsByClassName: Permite acceder a elementos que tienen una clase específica.
    getElementsByTagName: Permite acceder a elementos por su nombre de etiqueta.
    querySelector: Permite seleccionar un elemento utilizando un selector CSS.
    querySelectorAll: Permite seleccionar múltiples elementos que coinciden con un selector CSS.

  4. ¿Cómo se modifica un elemento del DOM?

    5. Para modificar un elemento del DOM, se pueden utilizar diferentes propiedades y métodos. Algunas acciones comunes incluyen cambiar el contenido de un elemento, modificar los atributos, agregar o eliminar clases, y manipular la estructura del árbol del DOM.

  5. ¿Cuál es la diferencia entre getElementby* y querySelector*?

    6. La diferencia entre getElementBy* y querySelector* radica en cómo se seleccionan los elementos. Los métodos getElementBy* toman como argumento una referencia específica, como el ID, la clase o la etiqueta, y devuelven una colección de nodos que coinciden con ese criterio. Por otro lado, querySelector* utiliza un selector CSS para seleccionar elementos y devuelve solo el primer elemento que coincide.

  6. ¿Qué es un evento?

    7. Un evento es una acción o suceso que ocurre en una página web, como hacer clic en un botón, cargar una página o mover el cursor del mouse. Los eventos permiten que el código JavaScript responda a las interacciones del usuario y realice acciones en consecuencia.

  7. ¿Qué es un listener?

    8. Un listener (o "escuchador") es una función que se adjunta a un elemento HTML para detectar y responder a un evento específico. Los listeners se utilizan para capturar eventos y ejecutar código en respuesta a ellos.

  8. ¿Qué es un callback?

    9. Un callback es una función que se pasa como argumento a otra función y se invoca en algún momento posterior. En el contexto de los eventos y los listeners, un callback se refiere a la función que se ejecuta cuando se produce un evento específico.

  9. ¿Qué es un objeto de evento?

    10. Un objeto de evento es un objeto que contiene información sobre un evento específico que ocurre en la página web. Proporciona detalles como el tipo de evento, el objetivo del evento, las coordenadas del puntero del mouse, las teclas presionadas, entre otros. El objeto de evento se pasa como argumento a la función de callback asociada al listener, lo que permite acceder a esta información y tomar acciones basadas en ella.
